# SV Lab — svlab.online

Marketing site for **SV Lab**, the solo AI product lab of **Siarhei Sheleh** (Sergey Sheleg) from Warsaw, Poland.
The site doubles as the portfolio for the lab's three live products (Prowl, CheckMyData, PrivateClawd), the career track record, the personal `/about` page, and the consulting front door at `/consulting` for AI advisory and AI transformation engagements. All inquiries land at [contact@svlab.online](mailto:contact@svlab.online).

- **Stack:** [Astro 6](https://astro.build) · TypeScript · Tailwind CSS v4
- **Hosting:** GitHub Pages (static) on the apex domain `svlab.online`
- **DNS:** Cloudflare
- **Author:** Siarhei Sheleh — `contact@svlab.online`

## Routes

| Route                               | Purpose                                                      |
| ----------------------------------- | ------------------------------------------------------------ |
| `/`                                 | Single-page lab overview: hero, about teaser, products, track record, capabilities, contact. |
| `/about`                            | Personal page for Siarhei Sheleh — portrait, bio, disciplines, selected work, CTA. Emits a `Person` JSON-LD schema. |
| `/consulting`                       | Consulting hub: positioning, two service cards, why-me strip, CTA. Emits a `ProfessionalService` JSON-LD schema. |
| `/consulting/ai-advisory`           | Landing page for AI advisory engagements (apply AI to a specific use case fast). Emits `Service` + `FAQPage` JSON-LD schemas. |
| `/consulting/ai-transformation`     | Landing page for AI transformation engagements (rewire processes, data, and org around AI). Emits `Service` + `FAQPage` JSON-LD schemas. |
| `/404`                              | Themed not-found page.                                       |

All "Book a consultation" buttons render as pre-filled `mailto:` links — no backend, deploys cleanly to GitHub Pages. Subject and body templates live in [`src/data/consulting.ts`](src/data/consulting.ts) (`service.cta`).

### Editing the About page

The full bio lives in [`src/components/AboutPage.astro`](src/components/AboutPage.astro). The home-page teaser lives in [`src/components/About.astro`](src/components/About.astro). Keep the two in sync when you change positioning copy.

The portrait is a placeholder SVG at [`public/about/portrait.svg`](public/about/portrait.svg). To swap in a real photo:

1. Drop a square JPG/PNG/WebP at `public/about/portrait.<ext>` (recommended 960×960, ≤ 200 KB).
2. Update the `<img src="/about/portrait.svg">` reference in [`src/components/AboutPage.astro`](src/components/AboutPage.astro) and the `image:` field in the `Person` JSON-LD schema in [`src/pages/about.astro`](src/pages/about.astro).
3. Remove the placeholder SVG (optional — it's harmless to leave behind).

### Editing consulting copy

All consulting page content (headlines, outcomes, proofs, process steps, FAQ, mailto subject/body, page meta) is centralized in [`src/data/consulting.ts`](src/data/consulting.ts). Edit one entry there to update both the corresponding landing page and any references on the `/consulting` hub. Each service references real `projects.ts` and `track-record.ts` slugs in `proofs.products` and `proofs.trackRecord` — make sure the slugs you list still exist.

### Adding a portfolio project

Edit `src/data/projects.ts` and append an entry:

```ts
{
  slug: "new-project",
  name: "New Project",
  domain: "newproject.com",
  url: "https://newproject.com",
  yearStart: 2026,
  yearEnd: "present",      // or a number for completed work
  status: "live",          // | "archived" | "in-development"
  category: "AI agents",
  tagline: "...",
  description: "...",
  highlights: ["...", "...", "..."],
  stack: ["TypeScript", "Postgres"],
  accent: "cyan",          // | "lime" | "violet"
}
```

The card and JSON-LD `SoftwareApplication` entry are generated automatically.

## Deployment

### 1) GitHub Pages

The site is built and published by `.github/workflows/deploy.yml` on every push to `main`. The workflow uses the official [`withastro/action`](https://github.com/withastro/action) and `actions/deploy-pages`.

After the first push:

```bash
# Source = "GitHub Actions"
gh api -X POST repos/ssheleg/svlab-online-web/pages -f build_type=workflow

# Custom domain + HTTPS enforcement (run after the first deploy completes)
gh api -X PUT repos/ssheleg/svlab-online-web/pages \
  -f cname=svlab.online -F https_enforced=true
```

The `public/CNAME` file is part of the build, so GitHub Pages keeps the custom domain across deploys.

### 2) Cloudflare DNS

In the `svlab.online` zone:

| Type    | Name           | Value                                                                                       | Proxy                              |
| ------- | -------------- | ------------------------------------------------------------------------------------------- | ---------------------------------- |
| A       | `@`            | `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153` (all four)       | DNS only at first, then Proxied    |
| AAAA    | `@` (optional) | `2606:50c0:8000::153`, `2606:50c0:8001::153`, `2606:50c0:8002::153`, `2606:50c0:8003::153`  | match the A records above          |
| CNAME   | `www`          | `ssheleg.github.io`                                                                         | DNS only at first, then Proxied    |

> Cloudflare also supports CNAME flattening at the apex, so a single `CNAME @ → ssheleg.github.io` works equally well in place of the four A records.

**Proxy guidance:** start with **DNS only (grey cloud)** for the apex and `www` so GitHub can issue a Let's Encrypt cert (typically <10 min). Once the GitHub Pages settings show "Your site is published at https://svlab.online" with a green padlock, switch the records back to **Proxied (orange cloud)** to get Cloudflare CDN/WAF in front.

**SSL/TLS mode:** in Cloudflare → SSL/TLS → Overview, set the encryption mode to **Full** (not "Flexible"). GitHub Pages already serves HTTPS; "Flexible" would create a redirect loop.

**Email:** the `mailto:contact@svlab.online` link relies on whatever MX records are already configured for the zone — this site does not change them.

## SEO + AI search

- `<title>` and meta description tuned for "AI product lab · Poland".
- OpenGraph + Twitter cards point at `/og.png` (auto-generated from `/og.svg`).
- JSON-LD `Organization` schema in the layout (founder, address, email, sameAs).

## Accessibility & motion

- Skip-to-content link.
- Animations honor `prefers-reduced-motion: reduce`.
- All interactive elements are reachable by keyboard with visible focus styles via the browser default plus a styled skip link.
- Color contrast on body copy and buttons targets WCAG AA on the dark background.
