# SV Lab — svlab.online

Marketing site for **SV Lab**, an AI product lab from Poland.
Public single-page site that doubles as the portfolio for the lab's three live products (Prowl, CheckMyData, PrivateClawd) and the front door for [contact@svlab.online](mailto:contact@svlab.online).

- **Stack:** [Astro 6](https://astro.build) · TypeScript · Tailwind CSS v4
- **Hosting:** GitHub Pages (static) on the apex domain `svlab.online`
- **DNS:** Cloudflare
- **Author:** Siarhei Sheleh — `contact@svlab.online`

---


### Adding a portfolio project

Edit `src/data/projects.ts` and append an entry:

```ts
{
  slug: "new-project",
  name: "New Project",
  domain: "newproject.com",
  url: "https://newproject.com",
  yearStart: 2026,
  yearEnd: "present",      // or a number for completed work
  status: "live",          // | "archived" | "in-development"
  category: "AI agents",
  tagline: "...",
  description: "...",
  highlights: ["...", "...", "..."],
  stack: ["TypeScript", "Postgres"],
  accent: "cyan",          // | "lime" | "violet"
}
```

The card and JSON-LD `SoftwareApplication` entry are generated automatically.

## Deployment

### 1) GitHub Pages

The site is built and published by `.github/workflows/deploy.yml` on every push to `main`. The workflow uses the official [`withastro/action`](https://github.com/withastro/action) and `actions/deploy-pages`.

After the first push:

```bash
# Source = "GitHub Actions"
gh api -X POST repos/ssheleg/svlab-online-web/pages -f build_type=workflow

# Custom domain + HTTPS enforcement (run after the first deploy completes)
gh api -X PUT repos/ssheleg/svlab-online-web/pages \
  -f cname=svlab.online -F https_enforced=true
```

The `public/CNAME` file is part of the build, so GitHub Pages keeps the custom domain across deploys.

### 2) Cloudflare DNS

In the `svlab.online` zone:

| Type    | Name           | Value                                                                                       | Proxy                              |
| ------- | -------------- | ------------------------------------------------------------------------------------------- | ---------------------------------- |
| A       | `@`            | `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153` (all four)       | DNS only at first, then Proxied    |
| AAAA    | `@` (optional) | `2606:50c0:8000::153`, `2606:50c0:8001::153`, `2606:50c0:8002::153`, `2606:50c0:8003::153`  | match the A records above          |
| CNAME   | `www`          | `ssheleg.github.io`                                                                         | DNS only at first, then Proxied    |

> Cloudflare also supports CNAME flattening at the apex, so a single `CNAME @ → ssheleg.github.io` works equally well in place of the four A records.

**Proxy guidance:** start with **DNS only (grey cloud)** for the apex and `www` so GitHub can issue a Let's Encrypt cert (typically <10 min). Once the GitHub Pages settings show "Your site is published at https://svlab.online" with a green padlock, switch the records back to **Proxied (orange cloud)** to get Cloudflare CDN/WAF in front.

**SSL/TLS mode:** in Cloudflare → SSL/TLS → Overview, set the encryption mode to **Full** (not "Flexible"). GitHub Pages already serves HTTPS; "Flexible" would create a redirect loop.

**Email:** the `mailto:contact@svlab.online` link relies on whatever MX records are already configured for the zone — this site does not change them.

## SEO + AI search

- `<title>` and meta description tuned for "AI product lab · Poland".
- OpenGraph + Twitter cards point at `/og.png` (auto-generated from `/og.svg`).
- JSON-LD `Organization` schema in the layout (founder, address, email, sameAs).
- One `SoftwareApplication` schema per portfolio entry, generated from `projects.ts`.
- `robots.txt` allows everything and points at `sitemap-index.xml` (built by `@astrojs/sitemap`).

## Accessibility & motion

- Skip-to-content link.
- Animations honor `prefers-reduced-motion: reduce`.
- All interactive elements are reachable by keyboard with visible focus styles via the browser default plus a styled skip link.
- Color contrast on body copy and buttons targets WCAG AA on the dark background.
